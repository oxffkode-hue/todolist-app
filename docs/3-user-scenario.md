**제품명**: TodoListApp  
**문서 버전**: 1.0  
**작성일**: 2026-05-13  
**작성자**: UX/비즈니스 분석팀

## 페르소나 개요

**김지현 (32세, 마케팅 팀장)**

- 중견 소비재 기업 마케팅팀 팀장
- 하루 평균 5~8개 미팅, 다수 프로젝트 병행 관리
- 오전 9시 출근, 오후 7~8시 퇴근
- 업무용 PC(Chrome) + 개인 스마트폰(iOS Safari) 병행 사용
- 기존에 메모 앱과 엑셀로 할일을 관리했으나 분산 문제로 통합 솔루션 필요

## 시나리오 1: 신규 사용자 등록

### 배경
김지현은 동료에게 TodoListApp을 추천받았다. 월요일 퇴근 후 집에서 PC로 접속해 계정을 만들려 한다. 기존에 쓰던 엑셀 파일이 너무 지저분해져서 새로운 도구를 시도해보기로 결심했다.

### 목표
이메일과 비밀번호로 계정을 생성하고 서비스를 사용할 수 있는 상태가 되는 것.

### 사용 환경
PC 웹 (Chrome, 1920x1080)

### 정상 흐름 (Happy Path)
1. 김지현이 TodoListApp URL에 접속하면 로그인 화면이 표시된다.
2. "회원가입" 링크를 클릭해 회원가입 폼으로 이동한다.
3. 이름 입력란에 "김지현"을 입력한다.
4. 이메일 입력란에 본인 이메일 주소를 입력한다.
5. 비밀번호 입력란에 영문+숫자 조합 8자 이상의 비밀번호를 입력한다.
6. 비밀번호 확인란에 동일한 비밀번호를 재입력한다.
7. "회원가입" 버튼을 클릭한다.
8. 서버가 이메일 중복 여부를 확인하고 계정을 생성한다.
9. 회원가입 완료 메시지가 표시되고 로그인 화면으로 이동한다.

### 예외 흐름 (Exception Path)
- **이미 가입된 이메일 입력**: "이미 사용 중인 이메일입니다" 오류 메시지 표시. 다른 이메일 입력 또는 로그인 화면으로 이동 안내.
- **비밀번호 8자 미만 입력**: "비밀번호는 8자 이상 영문과 숫자를 포함해야 합니다" 유효성 오류 표시. 제출 불가.
- **비밀번호 영문 또는 숫자 누락**: 동일한 유효성 오류 메시지 표시. 제출 불가.
- **비밀번호 확인 불일치**: "비밀번호가 일치하지 않습니다" 오류 표시. 제출 불가.
- **이름 또는 이메일 미입력**: 필수 항목 안내 메시지 표시. 제출 불가.
- **네트워크 오류**: "잠시 후 다시 시도해주세요" 오류 메시지 표시.

### 결과
김지현의 계정이 생성된다. 비밀번호는 bcrypt로 암호화되어 저장된다. 업무/개인/기타 기본 카테고리 3개가 계정에 자동 포함된다. 이후 로그인하여 서비스를 이용할 수 있다.

## 시나리오 2: 매일 아침 로그인과 업무 시작

### 배경
화요일 오전 9시, 김지현이 사무실에 출근해 PC를 켰다. 오늘 하루 처리할 업무를 파악하기 위해 TodoListApp에 로그인하고 오늘의 할일 목록을 확인하려 한다.

### 목표
로그인 후 오늘 처리해야 할 업무 카테고리 할일을 필터링하여 업무 우선순위를 파악하는 것.

### 사용 환경
PC 웹 (Chrome)

### 정상 흐름 (Happy Path)
1. TodoListApp에 접속하면 로그인 화면이 나타난다.
2. 등록한 이메일과 비밀번호를 입력하고 "로그인" 버튼을 클릭한다.
3. 서버가 인증 후 JWT 토큰을 발급하고 메인 화면(할일 목록)으로 이동한다.
4. 할일 목록 화면에서 카테고리 필터를 "업무"로 선택한다.
5. 완료 여부 필터를 "미완료"로 설정한다.
6. 종료예정일 필터를 "오늘 ~ 이번 주"로 설정한다.
7. AND 조건으로 필터링된 결과: 이번 주 내 마감인 미완료 업무 할일만 표시된다.
8. 목록을 훑어보며 오늘 집중할 항목을 파악한다.

### 예외 흐름 (Exception Path)
- **이메일 또는 비밀번호 오류**: "이메일 또는 비밀번호가 올바르지 않습니다" 메시지 표시. 재입력 요청.
- **가입하지 않은 이메일**: 동일한 오류 메시지 표시 (보안상 계정 존재 여부 미구분).
- **JWT 토큰 만료 상태에서 재접속**: 401 응답으로 로그인 화면으로 자동 리다이렉트.
- **필터 조건에 맞는 할일 없음**: "조건에 맞는 할일이 없습니다" 빈 상태 메시지 표시.

### 결과
김지현은 이번 주 마감인 미완료 업무 할일 목록을 확인해 오늘의 우선순위를 설정한다. 로그인 세션이 유지되어 이후 작업을 연속으로 진행할 수 있다.

## 시나리오 3: 점심시간 중 모바일로 할일 등록

### 배경
수요일 점심시간 12시 30분, 김지현이 구내식당에서 식사를 마치고 팀원에게 보고받은 신규 캠페인 관련 업무를 잊기 전에 기록하려 한다. PC 자리로 돌아가기 전이라 스마트폰으로 빠르게 입력하고자 한다.

### 목표
모바일 환경에서 새로운 할일을 카테고리와 함께 등록하고, 필요시 새 카테고리도 생성하는 것.

### 사용 환경
모바일 웹 (iOS Safari, iPhone)

### 정상 흐름 (Happy Path)
1. 스마트폰으로 TodoListApp에 접속한다. 이미 로그인 상태라면 곧바로 할일 목록 화면이 표시된다.
2. 화면 하단 또는 상단의 "할일 추가(+)" 버튼을 탭한다.
3. 할일 등록 폼이 표시된다.
4. 제목 입력란에 "SNS 캠페인 소재 기획안 초안 작성"을 입력한다.
5. 설명 입력란에 "팀장 검토 전 목요일 오전까지 초안 완성"을 입력한다.
6. 종료예정일을 내일 날짜(목요일)로 선택한다.
7. 카테고리 선택 드롭다운을 열었을 때 적합한 카테고리가 없어 새 카테고리 "캠페인"을 생성하기로 한다.
8. "카테고리 추가" 옵션을 탭하고 "캠페인"이라는 이름으로 카테고리를 생성한다.
9. 새로 생성된 "캠페인" 카테고리를 선택한다.
10. "저장" 버튼을 탭한다.
11. 할일이 등록되고 목록에 "캠페인" 카테고리 태그와 함께 표시된다.

### 예외 흐름 (Exception Path)
- **제목 미입력 후 저장 시도**: "제목은 필수 입력 항목입니다" 오류 표시. 저장 불가.
- **카테고리 미선택 후 저장 시도**: "카테고리를 선택해주세요" 오류 표시. 저장 불가.
- **중복 카테고리명 입력**: "이미 존재하는 카테고리명입니다" 오류 표시. 다른 이름 요청.
- **세션 만료 상태**: 401 응답으로 로그인 화면 이동. 재로그인 후 다시 시도 필요.
- **네트워크 불안정**: 저장 실패 안내 및 재시도 버튼 표시. 입력 내용은 유지.

### 결과
"SNS 캠페인 소재 기획안 초안 작성" 할일이 "캠페인" 카테고리로 등록된다. 종료예정일이 내일로 설정되어 마감 관리가 가능하다. "캠페인" 카테고리는 이후에도 재사용할 수 있다.

## 시나리오 4: 퇴근 전 할일 진행 상황 점검 및 완료 처리

### 배경
목요일 오후 6시, 김지현이 퇴근 준비를 하며 오늘 처리한 업무를 점검하려 한다. PC 앞에 앉아 오늘 날짜 기준으로 완료/미완료 항목을 확인하고, 완료된 할일을 처리하려 한다.

### 목표
오늘 완료한 할일을 완료 상태로 전환하고 내일 처리할 미완료 항목을 파악하는 것.

### 사용 환경
PC 웹 (Chrome)

### 정상 흐름 (Happy Path)
1. 할일 목록 화면에서 종료예정일 필터를 "오늘"로 설정한다.
2. 완료 여부 필터를 "전체"로 설정해 오늘 마감인 모든 항목을 확인한다.
3. "SNS 캠페인 소재 기획안 초안 작성" 항목이 목록에 표시된다.
4. 해당 항목 옆의 완료 체크박스 또는 "완료 처리" 버튼을 클릭한다.
5. 서버에서 `isCompleted: true`로 상태가 업데이트된다.
6. 해당 항목이 시각적으로 완료 스타일(취소선, 색상 변경 등)로 표시된다.
7. 나머지 미완료 항목을 확인하고 내일 처리할 업무를 머릿속으로 정리한다.
8. 퇴근 전 로그아웃 버튼을 클릭해 세션을 종료한다.
9. 서버에서 클라이언트 JWT 토큰이 삭제되고 로그인 화면으로 이동한다.

### 예외 흐름 (Exception Path)
- **타인의 할일에 완료 처리 시도**: 403 Forbidden 응답. 본인 소유 항목만 수정 가능 안내.
- **네트워크 오류로 상태 업데이트 실패**: 오류 메시지 표시. UI 상태는 이전으로 복원.
- **필터 조건에 맞는 항목 없음**: "오늘 마감인 할일이 없습니다" 빈 상태 메시지 표시.

### 결과
오늘 완료한 할일은 `isCompleted: true` 상태로 업데이트된다. 로그아웃으로 JWT 토큰이 클라이언트에서 삭제되어 보안이 유지된다. 김지현은 내일 처리할 항목을 명확히 파악하고 퇴근한다.

## 시나리오 5: 완료된 할일 되돌리기

### 배경
금요일 오전, 김지현이 어제 완료 처리한 "SNS 캠페인 소재 기획안 초안 작성"을 다시 확인했더니 상사의 피드백으로 추가 수정이 필요하다는 것을 알았다. 완료 상태를 취소하고 다시 미완료로 되돌려야 한다.

### 목표
완료 처리된 할일을 미완료 상태로 되돌려 다시 작업 대상으로 포함시키는 것.

### 사용 환경
PC 웹 (Chrome)

### 정상 흐름 (Happy Path)
1. 할일 목록 화면에서 완료 여부 필터를 "완료"로 설정한다.
2. 어제 완료 처리한 "SNS 캠페인 소재 기획안 초안 작성" 항목이 목록에 표시된다.
3. 해당 항목의 완료 체크박스를 다시 클릭(토글)한다.
4. 서버에서 `isCompleted: false`로 상태가 업데이트된다.
5. 해당 항목의 완료 스타일이 해제되고 미완료 상태로 표시 변경된다.
6. 완료 여부 필터를 "미완료"로 변경하면 해당 항목이 미완료 목록에 나타난다.

### 예외 흐름 (Exception Path)
- **완료 항목이 필터에서 보이지 않음**: 완료 여부 필터가 "미완료"로 설정되어 있는 경우. "전체" 또는 "완료"로 필터 변경 안내 필요.
- **상태 업데이트 서버 오류**: 오류 메시지 표시. UI는 이전 완료 상태로 복원.
- **타인 소유 항목 토글 시도**: 403 Forbidden 응답.

### 결과
"SNS 캠페인 소재 기획안 초안 작성"이 다시 `isCompleted: false` 상태로 변경된다. 미완료 목록에 복귀하여 수정 작업을 재개할 수 있다.

## 시나리오 6: 불필요한 할일 정리

### 배경
월말 정리 시간, 김지현이 이미 지나간 행사나 취소된 프로젝트 관련 할일들이 목록에 쌓여 있는 것을 확인했다. 더 이상 필요 없는 항목들을 삭제해 목록을 깔끔하게 정리하려 한다.

### 목표
완료되었거나 더 이상 필요 없는 할일을 삭제해 목록의 가독성을 높이는 것.

### 사용 환경
PC 웹 (Chrome)

### 정상 흐름 (Happy Path)
1. 할일 목록 화면에서 완료 여부 필터를 "완료"로 설정한다.
2. 완료된 할일 목록을 훑어보며 삭제할 항목을 선택한다.
3. 삭제할 항목의 "삭제" 버튼 또는 아이콘을 클릭한다.
4. "정말 삭제하시겠습니까?" 확인 다이얼로그가 표시된다.
5. "확인" 버튼을 클릭한다.
6. 서버에서 해당 할일이 삭제된다.
7. 목록에서 해당 항목이 즉시 제거된다.
8. 필요한 항목에 대해 3~7 단계를 반복한다.

### 예외 흐름 (Exception Path)
- **타인 소유 할일 삭제 시도**: 403 Forbidden 응답. "본인의 할일만 삭제할 수 있습니다" 메시지 표시.
- **삭제 확인 다이얼로그에서 "취소" 선택**: 삭제 취소. 항목이 목록에 유지됨.
- **서버 오류로 삭제 실패**: 오류 메시지 표시. 항목은 목록에 유지됨.
- **이미 삭제된 항목 재시도**: 404 응답. "항목을 찾을 수 없습니다" 메시지 표시.

### 결과
선택한 할일 항목들이 데이터베이스에서 완전히 삭제된다. 목록이 정리되어 현재 진행 중인 항목들만 남아 가독성이 향상된다.

## 시나리오 7: 카테고리 신규 추가 및 정리

### 배경
분기 초, 김지현의 팀에 새로운 프로젝트 "브랜드 리뉴얼"이 시작되었다. 기존 카테고리(업무/개인/기타/캠페인)로는 분류가 어려워 새 카테고리를 추가하려 한다. 또한 지난 분기에 만든 임시 카테고리 "이벤트"는 더 이상 사용하지 않아 삭제하려 한다.

### 목표
새 카테고리 "브랜드 리뉴얼"을 추가하고, 사용하지 않는 사용자 정의 카테고리 "이벤트"를 삭제하는 것.

### 사용 환경
PC 웹 (Chrome)

### 정상 흐름 (Happy Path)
1. 설정 또는 카테고리 관리 화면으로 이동한다.
2. 현재 카테고리 목록이 표시된다: 업무(기본), 개인(기본), 기타(기본), 캠페인(사용자 정의), 이벤트(사용자 정의).
3. "카테고리 추가" 버튼을 클릭한다.
4. 카테고리명 입력란에 "브랜드 리뉴얼"을 입력하고 저장한다.
5. "브랜드 리뉴얼" 카테고리가 목록에 추가된다.
6. "이벤트" 카테고리 옆의 "삭제" 아이콘을 클릭한다.
7. 확인 다이얼로그에서 "확인"을 클릭한다.
8. "이벤트" 카테고리가 목록에서 제거된다.

### 예외 흐름 (Exception Path)
- **기본 카테고리(업무/개인/기타) 수정 시도**: 수정 버튼 비활성화 또는 "기본 카테고리는 수정할 수 없습니다" 안내.
- **기본 카테고리 삭제 시도**: 삭제 버튼 비활성화 또는 "기본 카테고리는 삭제할 수 없습니다" 안내.
- **할일이 있는 카테고리 삭제 시도**: "해당 카테고리에 할일이 존재합니다. 할일을 먼저 삭제하거나 이동해주세요" 오류 표시. 삭제 불가.
- **중복 카테고리명 입력**: "이미 존재하는 카테고리명입니다" 오류 표시.
- **카테고리명 미입력 저장 시도**: "카테고리명을 입력해주세요" 오류 표시.

### 결과
"브랜드 리뉴얼" 카테고리가 새로 생성되어 이후 할일 등록 시 선택 가능하다. 할일이 없던 "이벤트" 카테고리는 삭제되어 목록이 정리된다. 기본 카테고리 3개는 변경 없이 유지된다.

## 시나리오 8: 개인정보(비밀번호) 변경

### 배경
보안 의식이 높은 김지현은 3개월마다 비밀번호를 변경하는 습관이 있다. 이번 달에도 비밀번호 변경 주기가 되어 PC에서 계정 설정 화면으로 이동해 비밀번호를 변경하려 한다.

### 목표
현재 비밀번호를 확인받은 후 새 비밀번호로 안전하게 변경하는 것.

### 사용 환경
PC 웹 (Chrome)

### 정상 흐름 (Happy Path)
1. 로그인 상태에서 계정 설정 또는 프로필 화면으로 이동한다.
2. "비밀번호 변경" 섹션을 찾아 클릭하거나 해당 폼을 펼친다.
3. "현재 비밀번호" 입력란에 기존 비밀번호를 입력한다.
4. "새 비밀번호" 입력란에 새로운 비밀번호(8자 이상, 영문+숫자 포함)를 입력한다.
5. "새 비밀번호 확인" 입력란에 동일한 새 비밀번호를 재입력한다.
6. "변경" 버튼을 클릭한다.
7. 서버에서 현재 비밀번호를 bcrypt로 검증한다.
8. 검증 통과 후 새 비밀번호를 bcrypt로 암호화하여 저장한다.
9. "비밀번호가 성공적으로 변경되었습니다" 성공 메시지가 표시된다.

### 예외 흐름 (Exception Path)
- **현재 비밀번호 오입력**: "현재 비밀번호가 올바르지 않습니다" 오류 표시. 재입력 요청.
- **새 비밀번호 유효성 미충족(8자 미만, 영문 또는 숫자 누락)**: 유효성 오류 메시지 표시. 제출 불가.
- **새 비밀번호 확인 불일치**: "새 비밀번호가 일치하지 않습니다" 오류 표시. 제출 불가.
- **현재 비밀번호와 동일한 새 비밀번호 입력**: "현재 비밀번호와 다른 비밀번호를 입력해주세요" 안내 (정책에 따라 선택적 적용).
- **인증 만료 상태**: 401 응답으로 로그인 화면 이동.

### 결과
새 비밀번호가 bcrypt로 암호화되어 저장된다. 다음 로그인 시 새 비밀번호를 사용해야 한다. 기존 세션은 유지되거나 보안 정책에 따라 재로그인을 요청할 수 있다.

## 시나리오 9: 서비스 탈퇴

### 배경
김지현이 회사를 이직하면서 새 직장에서 팀 공유 일정 도구를 도입하게 되었다. 개인 TodoListApp은 더 이상 사용하지 않을 것 같아 개인정보 보호 차원에서 계정을 완전히 삭제하기로 결정했다.

### 목표
비밀번호 확인 후 계정과 모든 데이터를 완전히 삭제하는 것.

### 사용 환경
PC 웹 (Chrome)

### 정상 흐름 (Happy Path)
1. 로그인 상태에서 계정 설정 화면으로 이동한다.
2. 화면 하단의 "회원 탈퇴" 링크 또는 버튼을 클릭한다.
3. 탈퇴 안내 페이지 또는 모달이 표시된다.
   - 안내 내용: "탈퇴 시 모든 할일, 카테고리, 계정 정보가 즉시 삭제되며 복구할 수 없습니다."
4. 비밀번호 확인 입력란에 현재 비밀번호를 입력한다.
5. "탈퇴 확인" 버튼을 클릭한다.
6. 최종 확인 다이얼로그: "정말 탈퇴하시겠습니까? 이 작업은 되돌릴 수 없습니다."
7. "확인"을 클릭한다.
8. 서버에서 비밀번호를 검증한다.
9. 검증 통과 후 CASCADE DELETE로 할일, 카테고리, 계정 정보가 즉시 삭제된다.
10. JWT 토큰이 무효화되고 로그인 화면으로 이동한다.
11. "탈퇴가 완료되었습니다. 이용해주셔서 감사합니다." 메시지 표시.

### 예외 흐름 (Exception Path)
- **비밀번호 오입력**: "비밀번호가 올바르지 않습니다" 오류 표시. 탈퇴 진행 불가.
- **최종 확인 다이얼로그에서 "취소" 선택**: 탈퇴 중단. 계정 설정 화면으로 복귀.
- **서버 오류로 삭제 실패**: "탈퇴 처리 중 오류가 발생했습니다. 잠시 후 다시 시도해주세요" 메시지 표시. 계정 및 데이터 유지.
- **인증 만료 상태에서 탈퇴 시도**: 401 응답으로 로그인 화면 이동. 재로그인 후 재시도 필요.

### 결과
김지현의 계정, 모든 할일, 사용자 정의 카테고리가 데이터베이스에서 즉시 완전 삭제된다. 기본 카테고리(업무/개인/기타)는 user_id=NULL인 시스템 공유 데이터이므로 삭제되지 않는다. 해당 이메일로 재가입은 가능하다. JWT 토큰이 무효화되어 로그인 화면으로 이동한다. 개인정보가 완전히 제거되어 데이터 보호 요건을 충족한다.
